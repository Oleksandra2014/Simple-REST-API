Project Assignment: Simple REST API

This API provides functionality for managing Users, Income, and Expenses.

Base API_URL

http://localhost:8000

Users Endpoints

GET /users - Retrieve all users

POST /users - Add a new user (requires JSON body with new user's data like name, username, email, etc.)

PUT /users/:id - Update an existing user by ID (requires JSON body with updated fields)

DELETE /users/:id - Delete a user by ID

Income Endpoints

GET /income - Retrieve all income records

POST /income - Add a new income (requires JSON body with new data like source and amount)

PUT /income/:id - Update an existing income record by ID

DELETE /income/:id - Delete an income record by ID

Expenses Endpoints

GET /expenses - Retrieve all expense records

POST /expenses - Add a new expense (requires JSON body with category and amount)

PUT /expenses/:id - Update an existing expense record by ID

DELETE /expenses/:id - Delete an expense record by ID

Request & Response Format

Example POST Request (JSON Body for Users)

{
    "name": "Harry Potter",
    "username": "HaPo",
    "email": "harry@potter.com"
}

Example Response

{
    "id": 1,
    "name": "Harry Potter",
    "username": "HaPo",
    "email": "harry@potter.com"
}

Error Handling

All responses follow a consistent structure for errors and successful requests:

{
    "error": "User was not found"
}
            
{
    "message": "New user has been created successfuly"
}
